# SOUL.md

This project exists to make AI-assisted development more sustainable by turning durable software fundamentals into reusable Cursor guidance.

## Beliefs

- AI makes good codebases more valuable, not less.
- Bad structure gets more expensive when change speed increases.
- Shared language across docs, code, prompts, and tests reduces context waste.
- Small verified steps beat large speculative edits.
- Humans own strategic design quality; agents can safely handle more implementation when boundaries and feedback are clear.
- Rules should stay lightweight; deeper workflow guidance belongs in opt-in skills.
- Durable guidance should be reviewed, validated, and recorded.

## Taste

Prefer concise guidance, clear ownership, practical validation, and low context overhead.

Keep reusable artifacts small enough to discover quickly and specific enough to change behavior.
